### 第四步：编写 Python 版本的流式输出

### 为什么需要流式输出（Streaming）？

当大模型生成长文本时，如果一次性等待所有文本生成完毕再返回，用户会面临长时间的白屏等待（差体验）。流式输出可以让大模型“生成一个字，返回一个字”，在前端实现类似“打字机”的丝滑效果。

在 `01_llm_basics/python/` 目录下新建一个文件 **`streaming.py`**，写入以下代码：

```python
import os
from pathlib import Path
from dotenv import load_dotenv
from openai import OpenAI

# 1. 加载根目录环境变量
env_path = Path(__file__).resolve().parent.parent.parent / '.env'
load_dotenv(dotenv_path=env_path)

# 2. 初始化客户端
client = OpenAI(
    api_key=os.getenv("DEEPSEEK_API_KEY"),
    base_url=os.getenv("DEEPSEEK_BASE_URL")
)

# 3. 发起流式对话请求 (注意添加 stream=True)
response = client.chat.completions.create(
    model="deepseek-chat",
    messages=[
        {"role": "user", "content": "请写一首关于秋天的四句小诗。"}
    ],
    stream=True # 👈 开启流式传输开关
)

print("AI 的流式回答：")

# 4. 循环遍历流式返回的数据块 (Chunk)
for chunk in response:
    # 注意：流式返回的属性是 delta（增量），而不是之前的 message
    content = chunk.choices[0].delta.content
    if content is not None:
        # end="" 阻止 print 自动换行
        # flush=True 强制控制台立即刷新输出，不要在缓冲区等待
        print(content, end="", flush=True)

print() # 诗歌生成完毕后，打印一个换行
```

---

#### 💡 核心知识点解析：
1. **`stream=True`**：大模型接口不会再返回一个完整的 JSON，而是变成了一个**生成器（Generator）**，我们可以持续迭代它。
2. **`delta` 代替 `message`**：在非流式请求中，文本在 `message.content` 里；而在流式请求中，每一次返回的只是一个碎片（Token），我们需要从 `delta.content` 中提取这个增量。

