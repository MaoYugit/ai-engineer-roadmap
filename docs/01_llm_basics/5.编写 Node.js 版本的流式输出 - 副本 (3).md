### 第五步：编写 Node.js (TypeScript) 版本的流式输出

在 `01_llm_basics/nodejs/` 目录下新建一个文件 **`streaming.ts`**，写入以下代码：

```typescript
import OpenAI from "openai";
import dotenv from "dotenv";
import path from "path";
import { fileURLToPath } from "url";

// 1. 加载根目录环境变量
const __filename = fileURLToPath(import.meta.url);
const __dirname = path.dirname(__filename);
dotenv.config({ path: path.resolve(__dirname, "../../.env") });

// 2. 初始化客户端
const client = new OpenAI({
  apiKey: process.env.DEEPSEEK_API_KEY,
  baseURL: process.env.DEEPSEEK_BASE_URL,
});

async function main() {
  try {
    // 3. 发起流式对话请求 (注意添加 stream: true)
    const stream = await client.chat.completions.create({
      model: "deepseek-chat",
      messages: [
        { role: "user", content: "请写一首关于秋天的四句小诗。" },
      ],
      stream: true, // 👈 开启流式传输开关
    });

    console.log("AI 的流式回答：");

    // 4. 使用 for await...of 异步遍历流式数据块 (Chunk)
    for await (const chunk of stream) {
      const content = chunk.choices[0]?.delta?.content || "";
      
      // 注意：在 Node.js 中，console.log 会强制换行。
      // 为了实现打字机效果，我们必须使用 process.stdout.write 直接写入输出流。
      process.stdout.write(content);
    }

    process.stdout.write("\n"); // 结束后手动输出一个换行
  } catch (error) {
    console.error("流式请求失败：", error);
  }
}

main();
```

---

#### 💡 核心知识点解析：
1. **`for await...of`**：这是 JavaScript/TypeScript 专门用来处理**异步迭代器（Async Iterator）**的语法。流式请求返回的 `stream` 是一个异步迭代器，每一次循环都会等待网络接收到下一个文本碎片。
2. **`process.stdout.write(content)`**：在前端你可能会直接拼接 DOM 字符串；但在 Node.js 终端开发中，传统的 `console.log` 每调用一次就会换行，必须用 `process.stdout.write` 才能保持在同一行连续输出。

在 `nodejs` 目录下运行：

```bash
npx tsx streaming.ts
```
