### 第六步：编写 Python 版本的工具调用

### 什么是工具调用（Tool Calling）？

大模型虽然聪明，但它不知道当前的实时信息（如今天的天气、股市、你数据库里的数据）。
**工具调用的本质是**：我们写一个本地函数（比如查天气的 Python 函数），然后把这个函数的**名字和功能描述（Schema）**告诉大模型。大模型在理解用户意图后，如果发现需要这个工具，它不会直接给出答案，而是返回一个“指令”：“请帮我执行 `get_weather(location='北京')` 这个函数，并把结果告诉我。”。我们运行函数后把结果喂给它，它再组织语言回答用户。

**请注意：大模型自己不运行你的代码，它只做“选择题”和“参数提取器”。真正的函数运行在你的本地。**

在 `01_llm_basics/python/` 目录下新建一个文件 **`tool_calling.py`**，写入以下完整流程代码：

```python
import os
import json
from pathlib import Path
from dotenv import load_dotenv
from openai import OpenAI

# 1. 初始化
env_path = Path(__file__).resolve().parent.parent.parent / '.env'
load_dotenv(dotenv_path=env_path)

client = OpenAI(
    api_key=os.getenv("DEEPSEEK_API_KEY"),
    base_url=os.getenv("DEEPSEEK_BASE_URL")
)

# 2. 定义我们本地的模拟天气函数（通常在这里写查询数据库或外部 API 的逻辑）
def get_weather(location: str):
    if "北京" in location:
        return json.dumps({"location": "北京", "temperature": "12°C", "condition": "晴朗"})
    elif "上海" in location:
        return json.dumps({"location": "上海", "temperature": "18°C", "condition": "多云"})
    else:
        return json.dumps({"location": location, "temperature": "未知", "condition": "阴天"})

# 3. 声明工具的描述（向大模型解释我们有什么工具，参数是什么样子的）
tools = [
    {
        "type": "function",
        "function": {
            "name": "get_weather",
            "description": "获取指定城市的实时天气",
            "parameters": {
                "type": "object",
                "properties": {
                    "location": {
                        "type": "string",
                        "description": "城市名称，例如：北京、上海",
                    }
                },
                "required": ["location"],
            },
        }
    }
]

# 4. 第一轮对话：把用户问题和工具列表发给大模型
messages = [{"role": "user", "content": "北京今天天气怎么样？"}]

print("正在发送第一轮请求...")
response = client.chat.completions.create(
    model="deepseek-chat",
    messages=messages,
    tools=tools,       # 👈 告诉大模型，你有哪些工具可以使用
    tool_choice="auto" # 让大模型自主决定是否需要调用工具
)

response_message = response.choices[0].message
tool_calls = response_message.tool_calls

# 5. 判断大模型是否决定调用工具
if tool_calls:
    print("大模型决定调用工具！")
    # 将大模型的“调用工具意图”作为一条历史消息，追加到对话数组中（必须）
    messages.append(response_message)
    
    # 遍历需要执行的工具调用
    for tool_call in tool_calls:
        function_name = tool_call.function.name
        function_args = json.loads(tool_call.function.arguments)
        print(f"大模型要求调用的函数名: {function_name}")
        print(f"大模型提取出来的参数: {function_args}")
        
        # 6. 本地执行对应的 Python 函数
        if function_name == "get_weather":
            tool_output = get_weather(location=function_args.get("location"))
            print(f"本地函数执行结果: {tool_output}")
            
            # 7. 将工具执行的具体结果，作为一个新的角色 "tool" 追加到历史消息中
            messages.append({
                "role": "tool",
                "tool_call_id": tool_call.id, # 必须携带此 ID，让大模型知道这是对应哪个请求的答案
                "name": function_name,
                "content": tool_output
            })
            
    # 8. 第二轮对话：把带有工具结果的完整上下文，再次发送给大模型
    print("正在发送第二轮请求（包含工具执行结果）...")
    second_response = client.chat.completions.create(
        model="deepseek-chat",
        messages=messages
    )
    print("\nAI 结合工具最终回答：")
    print(second_response.choices[0].message.content)
else:
    print("大模型直接回答，未调用工具：")
    print(response_message.content)
```

---

#### 💡 核心知识点解析（这个流程极为关键，请仔细阅读）：
1. **完整的 Tool Calling 共有 2 轮请求**。
   - **第一轮**：用户提问  -->  大模型分析 --> 发现要调用工具 --> 返回 `tool_calls`（含参数和 ID）。
   - **本地执行**：你的 Python 代码解析参数并运行本地函数，得到输出。
   - **第二轮**：你的代码将“大模型的调用意图”和“工具的执行结果”合在一起，再次发给大模型 --> 大模型阅读结果，组织自然语言回复用户。
2. **`messages` 数组的历史演进**：
   - 第一轮前：`[ {user: ...} ]`
   - 第二轮前：`[ {user: ...}, {assistant (包含 tool_calls): ...}, {tool: (包含 output 和 ID): ...} ]`
