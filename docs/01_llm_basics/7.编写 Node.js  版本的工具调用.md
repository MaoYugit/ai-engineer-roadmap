### 第七步：编写 Node.js (TypeScript) 版本的工具调用

在这部分的 TypeScript 代码中，我们将引入官方 SDK 的**强类型支持**，这能极大地提升在前端或后端开发时的代码安全度。

在 `01_llm_basics/nodejs/` 目录下新建一个文件 **`tool_calling.ts`**，写入以下代码：

```typescript
import OpenAI from "openai";
import dotenv from "dotenv";
import path from "path";
import { fileURLToPath } from "url";

// 1. 加载环境变量
const __filename = fileURLToPath(import.meta.url);
const __dirname = path.dirname(__filename);
dotenv.config({ path: path.resolve(__dirname, "../../.env") });

// 2. 初始化客户端
const client = new OpenAI({
  apiKey: process.env.DEEPSEEK_API_KEY,
  baseURL: process.env.DEEPSEEK_BASE_URL,
});

// 3. 定义我们本地的模拟天气函数
function getWeather(location: string): string {
  if (location.includes("北京")) {
    return JSON.stringify({ location: "北京", temperature: "12°C", condition: "晴朗" });
  } else if (location.includes("上海")) {
    return JSON.stringify({ location: "上海", temperature: "18°C", condition: "多云" });
  } else {
    return JSON.stringify({ location: location, temperature: "未知", condition: "阴天" });
  }
}

// 4. 声明工具的描述（声明强类型，防止 Schema 写错）
const tools: OpenAI.Chat.ChatCompletionTool[] = [
  {
    type: "function",
    function: {
      name: "get_weather",
      description: "获取指定城市的实时天气",
      parameters: {
        type: "object",
        properties: {
          location: {
            type: "string",
            description: "城市名称，例如：北京、上海",
          },
        },
        required: ["location"],
      },
    },
  },
];

async function main() {
  try {
    // 5. 第一轮对话：向大模型提供用户问题和工具列表（声明消息队列的强类型）
    const messages: OpenAI.Chat.ChatCompletionMessageParam[] = [
      { role: "user", content: "北京今天天气怎么样？" },
    ];

    console.log("正在发送第一轮请求...");
    const response = await client.chat.completions.create({
      model: "deepseek-chat",
      messages: messages,
      tools: tools,
      tool_choice: "auto",
    });

    const responseMessage = response.choices[0].message;
    const toolCalls = responseMessage.tool_calls;

    // 6. 判断大模型是否决定调用工具
    if (toolCalls && toolCalls.length > 0) {
      console.log("大模型决定调用工具！");
      
      // 将第一轮大模型的返回追加到对话历史中
      messages.push(responseMessage);

      // 遍历工具调用
      for (const toolCall of toolCalls) {
        const functionName = toolCall.function.name;
        // 注意：大模型返回的 arguments 是一个 string 化的 JSON，需要解析
        const functionArgs = JSON.parse(toolCall.function.arguments);
        
        console.log(`大模型要求调用的函数名: ${functionName}`);
        console.log(`大模型提取出来的参数:`, functionArgs);

        if (functionName === "get_weather") {
          // 7. 执行本地的 TypeScript 函数
          const toolOutput = getWeather(functionArgs.location);
          console.log(`本地函数执行结果: ${toolOutput}`);

          // 8. 将工具输出作为一个新的 "tool" 角色追加到消息历史中
          messages.push({
            role: "tool",
            tool_call_id: toolCall.id, // 必须传入相对应的 tool_call_id
            content: toolOutput,
          });
        }
      }

      // 9. 第二轮对话：再次发送给大模型，获取结合工具结果后的自然语言回答
      console.log("正在发送第二轮请求（包含工具执行结果）...");
      const secondResponse = await client.chat.completions.create({
        model: "deepseek-chat",
        messages: messages,
      });

      console.log("\nAI 结合工具最终回答：");
      console.log(secondResponse.choices[0].message.content);
    } else {
      console.log("大模型直接回答，未调用工具：");
      console.log(responseMessage.content);
    }
  } catch (error) {
    console.error("请求失败：", error);
  }
}

main();
```

---

#### 💡 核心知识点解析：
1. **强类型参数：`OpenAI.Chat.ChatCompletionMessageParam[]`**
   - 官方 SDK 提供的这个联合类型包含了 `{ role: 'user' }`、`{ role: 'assistant' }`、`{ role: 'tool' }` 等所有合法的消息结构。在 Node.js 开发中，使用强类型能帮你自动补全并校验每一个角色所需的参数（比如当 `role: "tool"` 时，TypeScript 会强制要求你必须填入 `tool_call_id` 属性）。
2. **`JSON.parse` 处理参数**：大模型预测并生成的参数始终是**字符串形式的 JSON**（如 `"{\"location\": \"北京\"}"`），因此在传入本地 TS 函数前，必须经过 `JSON.parse` 的转化。
